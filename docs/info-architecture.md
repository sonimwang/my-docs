# 信息架构基础

## 什么是信息架构

信息架构 = 文档怎么组织、分类、导航，让读者快速找到需要的内容。

就像图书馆：小说按作者排、工具书按主题排、有索引卡帮你检索。文档站同理。

## 文档组织的三个原则

### 1. 按任务分类（不按功能分类）

| 反例 | 正例|
|------|----|
| "按钮 A 的说明" | "如何创建一个项目" |
| "接口 /user 的参数" | "获取用户信息" |

读者关心的是"我要做什么"，不是"系统有什么功能"。

### 2. 层级不超过 3 层

```
一级：快速开始
  二级：安装
    三级：Windows 安装  ← 最深到这
```

超过 3 层就读丢了，没人能找到。

### 3. 每个页面只讲一件事

- 页面标题 = 明确的问题或任务
- 不要在一个页面里"顺便"塞不相关的内容

## 一个文档站的典型结构

```
首页          → 我是谁 / 这是什么项目
快速开始      → 5 分钟能跑起来
教程          → 分场景、分步骤
API 参考      → 每个接口一页
常见问题      → 按主题排列
```
